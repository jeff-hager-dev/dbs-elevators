The Challenge

It turns out it is harder than you might think. The coding challenge presented is to develop an algorithim that given a set of elevator requests can schedule them in the most effecient manner. The team that can deliver all passengers in the least amount of time is the winner.

Description & Data Format

On the Data page you will be provided with JSON describing a building, elevators and requests by people moving from floor to floor.

The JSON for each challenge will look like this: 

             {
                 "challengeId": 1,
                 "challengeDesc": "normal day traffic",
                 "isActive": 1,
                 "minFloor": 1,
                 "maxFloor": 12,
                 "numberOfElevators": 3,
                 "maxCapacity": 15,
                 "secondsPerFloor": 2,
                 "secondsPerFloorOverTenFloors": 1,
                 "timeToOpenDoor": 5,
                 "timeToCloseDoor": 5,
                 "calls": [{
                     "callId": 1,
                     "callTime": 43,
                     "startFloor": 1,
                     "endFloor": 11
                 }, ]
             }

The metadata describing the elevators should be self explanatory. The array of calls represent the buttons being pushed by people througout the day. The callId is a unique key. The callTime is the time in seconds from the start of the time period. The start floor and end floors determine the trip.

You must provide back data in this format: 

            {
                "challengeId": 1, 
                "stops": 
                  [
                    {
                      "StopId": Number,
                      "ElevatorId": Number,
                      "Floor": Number,
                      "Pickup": [Number],
                      "DropOff": [Number]
                    },
                  ]
            }

The StopId is a unique key that specifies the order in which the elevator will move. The ElevatorId is the number of the elevator (1-N) as specified in numberOfElevators provided in the input dataset. The Floor indicates where the elevator has been moved to. Pickup is an array of callId, indicating which passengers are being picked up on that floor. Dropoff is of course then an array of callId where passengers have been delivered.

Your program must generate an output file that picks up and then drops off every passenger without exceeding elevator max capacity. Be careful not to tie your design too closely to the initial dataset, as the other datasets may simulate different scenarios.

The dataset will be submitted by uploading on the Submit page. After you submit you will receive details on how your score was calculated or details on why your upload did not pass vaidation. You can always check where your team stands on the leaderboard page.

For example, consider a simple example with an input of: 

                {
                  "callId": 416,
                  "callTime": 1912,
                  "startFloor": 2,
                  "endFloor": 7
                }

Let's suppose elevator 2 was last moved to floor 5 and your program chooses this elevator to answer the call. We will assume 2 seconds of travel per floor and 5 seconds to open/close and 8 seconds to load and that your program produced the following: 

                {
                  "StopId": n,
                  "ElevatorId": 2,
                  "Floor": 5,
                  "Pickup": null,
                  "Dropoff": null
                },  
                {
                  "StopId": n + 1,
                  "ElevatorId": 2,
                  "Floor": 2,
                  "Pickup": [416],
                  "Dropoff": null
                },
                {
                  "StopId": n + 2,
                  "ElevatorId": 2,
                  "Floor": 7,
                  "Pickup": null,
                  "Dropoff": [416]
                },

This would be scored at 39 seconds: 1) Move floor 5 to 2 (6 seconds) 2) Open door on 2 (5 seconds) 3) Load on 2 (8 seconds) 4) Close door on 2 (5 seconds) 5) Move floor 2 to 7 (10 seconds) 6) Open door on 7 (5 seconds)
